## todoist

Bidirectional sync between Todoist and local SQLite/markdown files.

| Command                                           | Description                                                       |
| ------------------------------------------------- | ----------------------------------------------------------------- |
| `todoist init`                                    | Create the database, config file, and directory structure         |
| `todoist pull`                                    | Pull tasks and projects from Todoist, applying filter rules       |
| `todoist push`                                    | Push local changes back to Todoist                                |
| `todoist sync`                                    | Run pull then push sequentially                                   |
| `todoist status`                                  | Show sync state: last sync time, pending local changes            |
| `todoist diff`                                    | Show differences between local DB and last pull                   |
| `todoist export`                                  | Export current DB state to structured markdown files              |
| `todoist config`                                  | Validate and display current YAML configuration                   |
| `todoist complete TASK_ID`                        | Mark a task as completed in Todoist                               |
| `todoist create CONTENT`                          | Create a new task in Todoist and store it locally                 |
| `todoist create CONTENT --due DATE_STRING`        | Create a task with a natural language due date (e.g. 'tomorrow')  |
| `todoist create CONTENT --priority {1,2,3,4}`     | Create a task with a priority level (1=normal, 4=urgent)          |
| `todoist create CONTENT --label LABEL_NAME`       | Create a task with one or more labels (repeatable)                |
| `todoist create CONTENT --section SECTION_NAME`   | Create a task in a specific section within the project            |
| `todoist create CONTENT --description TEXT`       | Create a task with a longer description                           |
| `todoist attach TASK_ID FILE_PATH`                | Upload a local file and attach it to a task as a comment          |
| `todoist attach TASK_ID FILE_PATH --message TEXT` | Attach with a custom comment message (default: "File attachment") |
| `todoist --dry-run pull`                          | Show what would be pulled without writing to DB                   |
| `todoist --dry-run push`                          | Show what would be pushed without calling the API                 |
| `todoist --project "Name" pull`                   | Limit pull to a single project                                    |
| `todoist --verbose pull`                          | Pull with verbose logging                                         |
| `todoist --config PATH pull`                      | Use an alternate config file                                      |

## web

Rule-based web crawler that converts pages to LLM-optimized markdown.

| Command                             | Description                                                |
| ----------------------------------- | ---------------------------------------------------------- |
| `web init`                          | Scaffold configs, output directories, and database         |
| `web config`                        | Validate and display the current configuration             |
| `web crawl --input URL`             | Crawl a single URL                                         |
| `web crawl --input-file PATH`       | Crawl URLs from a text file (direct mode)                  |
| `web crawl --batch`                 | Process all batch YAML configs in src/config/web/          |
| `web crawl --mode MODE`             | Crawl mode: blog, documentation, or direct (default: blog) |
| `web crawl --output PATH`           | Override the output directory from configuration           |
| `web crawl --max-depth INT`         | Maximum link depth to follow in documentation mode         |
| `web crawl --max-pages INT`         | Maximum number of pages to crawl                           |
| `web crawl --js-wait FLOAT`         | Seconds to wait after page load for JavaScript to render   |
| `web crawl --browser-channel TEXT`  | Playwright browser channel (e.g., 'chrome', 'msedge')      |
| `web crawl --stealth`               | Enable stealth mode (random user-agent and viewport)       |
| `web crawl --include-pattern REGEX` | Regex pattern URLs must match to be crawled (repeatable)   |
| `web crawl --exclude-pattern REGEX` | Regex pattern URLs matching this are skipped (repeatable)  |
| `web crawl --retry-attempts INT`    | Number of retry attempts on fetch failure                  |
| `web crawl --retry-delay FLOAT`     | Seconds to wait between retry attempts                     |
| `web crawl --extract-images`        | Download images found on crawled pages                     |
| `web crawl --dry-run`               | Preview what would be crawled without writing              |
| `web crawl --force`                 | Re-crawl URLs that were previously crawled successfully    |

## file-txt

Converts PDF, Office, and email files to markdown.

| Command                                           | Description                                                               |
| ------------------------------------------------- | ------------------------------------------------------------------------- |
| `file-txt init`                                   | Write default config and create the output directory                      |
| `file-txt config`                                 | Validate and display current configuration                                |
| `file-txt convert PATH`                           | Convert files to markdown (default operation)                             |
| `file-txt convert PATH --output PATH`             | Write output to a specific directory                                      |
| `file-txt convert PATH --llm`                     | Also generate llms-full.txt and llms.txt aggregate files                  |
| `file-txt convert PATH --include-page-numbers`    | Retain page number markers in PDF output                                  |
| `file-txt convert PATH --no-include-page-numbers` | Strip page number markers from PDF output                                 |
| `file-txt convert PATH --extract-images`          | Extract embedded images to img/ subdirectories                            |
| `file-txt convert PATH --no-extract-images`       | Skip image extraction                                                     |
| `file-txt convert PATH --prefer-html`             | For email files: prefer HTML body (converted to markdown) over plain text |
| `file-txt convert PATH --full-headers`            | Include additional email headers beyond From/To/Date                      |
| `file-txt convert PATH --include-attachments`     | Include attachment metadata in email output                               |
| `file-txt convert PATH --no-include-attachments`  | Exclude attachment metadata from email output                             |
| `file-txt convert PATH --nuke`                    | Delete source files after successful conversion                           |
| `file-txt convert PATH --dry-run`                 | Preview what would be converted without writing                           |

## gmail

Collects emails based on Gmail search rules, with optional Gemini AI extraction.

> **Shared Google auth:** `gmail auth`, `gcal auth`, and `gdrive auth` all write to the same `deep-thought-google` keychain entry and request combined Gmail + Calendar + Drive scopes. Running the auth command on any one of the three tools authenticates all three.

| Command                    | Description                                                 |
| -------------------------- | ----------------------------------------------------------- |
| `gmail init`               | Create the database, config file, and directory structure   |
| `gmail config`             | Validate and display current configuration                  |
| `gmail auth`               | Run OAuth 2.0 authorization flow (shared token — covers all three Google tools) |
| `gmail`                    | Run collection (default operation)                          |
| `gmail send PATH`          | Send an email composed from a markdown file                 |
| `gmail --dry-run`          | Preview what would be collected without writing             |
| `gmail --rule NAME`        | Run only the named rule (default: all rules)                |
| `gmail --force`            | Clear state and reprocess all matching emails               |
| `gmail --max-emails INT`   | Max emails to process per run                               |
| `gmail --output PATH`      | Override the output directory from configuration            |
| `gmail --save-config PATH` | Write a default example configuration file to PATH and exit |

## gcal

Pulls Google Calendar events and supports creating, updating, and deleting events.

> **Shared Google auth:** `gcal auth`, `gmail auth`, and `gdrive auth` all write to the same `deep-thought-google` keychain entry and request combined Gmail + Calendar + Drive scopes. Running the auth command on any one of the three tools authenticates all three.

| Command                      | Description                                                      |
| ---------------------------- | ---------------------------------------------------------------- |
| `gcal init`                  | Create config file and directory structure                       |
| `gcal config`                | Validate and display current configuration                       |
| `gcal auth`                  | Run OAuth 2.0 authorization flow (shared token — covers all three Google tools) |
| `gcal pull`                  | Pull events from Google Calendar                                 |
| `gcal create PATH`           | Create a calendar event from a markdown file                     |
| `gcal update PATH`           | Update a calendar event from a markdown file                     |
| `gcal delete EVENT_ID`       | Delete a calendar event                                          |
| `gcal pull --dry-run`        | Preview what would be pulled without writing                     |
| `gcal pull --force`          | Clear sync state and re-pull all events from scratch             |
| `gcal pull --days-back INT`  | Number of days back to include (overrides config)                |
| `gcal pull --days-ahead INT` | Number of days ahead to include (overrides config)               |
| `gcal --calendar ID pull`    | Comma-separated calendar IDs to target (default: all configured) |
| `gcal --output PATH pull`    | Override the output directory from configuration                 |
| `gcal --save-config PATH`    | Write a default example configuration file to PATH and exit      |

## search / research

AI-powered web research via the Perplexity API. Two separate entry points.

| Command                             | Description                                                   |
| ----------------------------------- | ------------------------------------------------------------- |
| `search init`                       | Create config file                                            |
| `search config`                     | Validate and display current configuration                    |
| `research init`                     | Create data directories and starter configuration             |
| `research config`                   | Validate and display current configuration                    |
| `search "query"`                    | Fast synchronous search (sonar model)                         |
| `search "query" --quick`            | Print answer to stdout without writing a file                 |
| `search "query" --output PATH`      | Write result to a specific file                               |
| `search "query" --context PATH`     | Include a local file as prior context (repeatable)            |
| `search "query" --domains TEXT`     | Restrict or exclude domains (prefix with - to exclude)        |
| `search "query" --recency WINDOW`   | Filter by recency: hour/day/week/month/3 months/6 months/year |
| `search "query" --dry-run`          | Preview what would be submitted without making API calls      |
| `research "query"`                  | Deep async research job (sonar-deep-research)                 |
| `research "query" --output PATH`    | Write result to a specific file                               |
| `research "query" --context PATH`   | Include a local file as prior context (repeatable)            |
| `research "query" --domains TEXT`   | Restrict or exclude domains (prefix with - to exclude)        |
| `research "query" --recency WINDOW` | Filter by recency: hour/day/week/month/3 months/6 months/year |
| `research "query" --dry-run`        | Preview what would be submitted without making API calls      |

## audio

Transcribes audio files to markdown using Whisper models.

| Command                                                                     | Description                                                |
| --------------------------------------------------------------------------- | ---------------------------------------------------------- |
| `audio init`                                                                | Create config file, output directory, and database         |
| `audio config`                                                              | Validate and display current configuration                 |
| `audio transcribe`                                                          | Transcribe audio files from the configured input directory |
| `audio transcribe --input PATH`                                             | Transcribe a specific file or directory                    |
| `audio transcribe --output PATH`                                            | Write transcripts to a specific directory                  |
| `audio transcribe --engine {mlx,whisper,auto}`                              | Transcription engine (auto selects MLX on Apple Silicon)   |
| `audio transcribe --model {tiny,base,small,medium,large-v3,large-v3-turbo}` | Whisper model size                                         |
| `audio transcribe --language TEXT`                                          | Force a language code (e.g., en, fr)                       |
| `audio transcribe --output-mode {paragraph,segment,timestamp}`              | Transcript format                                          |
| `audio transcribe --diarize`                                                | Enable speaker identification                              |
| `audio transcribe --no-diarize`                                             | Disable speaker identification                             |
| `audio transcribe --pause-threshold SECS`                                   | Pause duration for paragraph breaks                        |
| `audio transcribe --remove-fillers`                                         | Strip filler words from transcript                         |
| `audio transcribe --no-remove-fillers`                                      | Disable filler word removal                                |
| `audio transcribe --llm`                                                    | Generate llms.txt and llms-full.txt aggregate files        |
| `audio transcribe --nuke`                                                   | Delete input files after successful processing             |
| `audio transcribe --dry-run`                                                | Preview transcription without writing                      |
| `audio transcribe --force`                                                  | Reprocess files even if already in DB                      |

## gdrive

Incremental backup of local files to Google Drive.

> **Shared Google auth:** `gdrive auth`, `gmail auth`, and `gcal auth` all write to the same `deep-thought-google` keychain entry and request combined Gmail + Calendar + Drive scopes. Running the auth command on any one of the three tools authenticates all three.

| Command                     | Description                                                               |
| --------------------------- | ------------------------------------------------------------------------- |
| `gdrive init`               | Create the database and data directory for first use                      |
| `gdrive config`             | Validate and display current configuration                                |
| `gdrive auth`               | Run OAuth 2.0 browser consent flow (shared token — covers all three Google tools) |
| `gdrive status`             | Show counts of backed-up, updated, skipped, and failed files              |
| `gdrive`                    | Run the backup (default operation)                                        |
| `gdrive --dry-run`          | Preview what would change without uploading                               |
| `gdrive --force`            | Clear all cached state and re-upload all files from scratch               |
| `gdrive --prune`            | Delete Drive files whose local paths match any configured exclude_pattern |
| `gdrive --verbose`          | Enable debug-level log output                                             |
| `gdrive --save-config PATH` | Write a default example configuration file to PATH and exit               |

## secrets

Manage secrets in macOS Keychain for all deep-thought tools.

| Command                                   | Description                                    |
| ----------------------------------------- | ---------------------------------------------- |
| `secrets status`                          | Show where each known secret is stored         |
| `secrets migrate`                         | Migrate all .env secrets to Keychain           |
| `secrets set TOOL KEY_NAME`               | Store a secret in Keychain (prompts for value) |
| `secrets set TOOL KEY_NAME --value VALUE` | Store a secret in Keychain with explicit value |
| `secrets delete TOOL KEY_NAME`            | Remove a secret from Keychain                  |

## pytest

A Python testing framework that makes it easy to write and run small, readable tests.

| Command                  | Description                                       |
| ------------------------ | ------------------------------------------------- |
| `pytest`                 | Run all tests in the configured test paths        |
| `pytest -k "expression"` | Run only tests matching the given name expression |
| `pytest -m "marker"`     | Run only tests matching the given marker          |
| `pytest -x`              | Stop on first failure                             |
| `pytest --maxfail=N`     | Stop after N failures                             |
| `pytest --fixtures`      | Show available fixtures                           |
| `pytest --markers`       | Show available markers                            |
| `pytest --co`            | Collect and display tests without running them    |

## pytest-cov

A pytest plugin that measures code coverage during test runs and generates reports.

| Command                            | Description                                   |
| ---------------------------------- | --------------------------------------------- |
| `pytest --cov=src`                 | Run tests with coverage for the src directory |
| `pytest --cov-report=term-missing` | Show missing lines in terminal output         |
| `pytest --cov-report=html`         | Generate an HTML coverage report              |
| `pytest --cov-report=xml`          | Generate an XML coverage report               |
| `pytest --cov-report=json`         | Generate a JSON coverage report               |
| `pytest --no-cov`                  | Disable coverage for a test run               |

## ruff

An extremely fast Python linter and code formatter, replacing tools like flake8 and isort.

| Command                 | Description                                    |
| ----------------------- | ---------------------------------------------- |
| `ruff check .`          | Lint all Python files in the current directory |
| `ruff check --fix .`    | Lint and auto-fix fixable issues               |
| `ruff format .`         | Format all Python files                        |
| `ruff format --check .` | Check formatting without making changes        |
| `ruff rule RULE`        | Explain a specific lint rule                   |
| `ruff linter`           | List all supported upstream linters            |
| `ruff clean`            | Clear ruff caches                              |
| `ruff config`           | List available configuration options           |

## mypy

A static type checker for Python that catches type errors before code runs.

| Command                              | Description                                |
| ------------------------------------ | ------------------------------------------ |
| `mypy src/`                          | Type check the src directory               |
| `mypy -m module`                     | Type check a specific module               |
| `mypy -p package`                    | Type check a specific package              |
| `mypy --strict src/`                 | Run with all strict checking flags enabled |
| `mypy --ignore-missing-imports src/` | Skip errors for missing third-party stubs  |
| `mypy --show-error-codes src/`       | Display error codes alongside messages     |

## prettier

An opinionated code formatter for JSON, Markdown, YAML, and other non-Python files.

| Command                           | Description                                     |
| --------------------------------- | ----------------------------------------------- |
| `npm run format`                  | Format all matched files in place               |
| `npm run format:check`            | Check if files are formatted without writing    |
| `npx prettier --write "path"`     | Format a specific file or glob pattern          |
| `npx prettier --check "path"`     | Check formatting for a specific file or glob    |
| `npx prettier --list-different .` | List files that differ from expected formatting |
